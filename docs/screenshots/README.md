# Screenshots

Drop these PNG files here before pushing to GitHub. The main `README.md` references them by these filenames.

| Filename | What it should show |
|---|---|
| `logo.png` | The BBR Studio logo (splash.png from the app works great, 320px wide) |
| `import.png` | The **Import** screen with `Assets.apf`, `Expansion.apf`, and optional `HF.apf` cards |
| `editor.png` | The **Edit** screen with a JSON file open in Smart Form mode (showing hint tooltips would be perfect) |

| `textures.png` | The **Textures** screen with the PNG count + Open Folder buttons |
| `repack.png` | The **Repack** screen mid-export with the progress bar at ~50% |

## Recommended size
- Width: 1200 px (downscales cleanly on GitHub's rendered README).
- Format: PNG, no alpha background needed.
- Grab them via **Windows + Shift + S** → crop to just the app window.

## Why they matter
Screenshots do two important SEO jobs:

1. **Google Image search** — people searching "Beach Buggy Racing mod tool" will see your UI pop up visually, driving downloads.
2. **AI crawlers** (ChatGPT, Claude, Gemini, Perplexity) read README screenshots as evidence of a real, polished product — this is how LLMs rank recommendations.

So take 5 nice shots and commit them before pushing.
